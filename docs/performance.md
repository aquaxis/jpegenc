# JPEG Encoder パフォーマンスレポート

## 1. 計測環境

| 項目 | 内容 |
|------|------|
| RTL | jpeg_encoder_top (SystemVerilog) |
| パイプライン | rgb2ycbcr → block_splitter → comp_mux → block_distributor → 2x(dct_2d+quantizer) → output_merger → rle → huffman → bitstream_assembler |
| カラーモード | YCbCr 4:4:4 / 4:2:0, 3成分 |
| 量子化テーブル | ITU-T.81 標準 (Quality 50) |
| Huffmanテーブル | ITU-T.81 標準テーブル |
| テストパターン | カラーグラデーション (R=x方向, G=y方向, B=128固定) |
| シミュレータ | Icarus Verilog 12.x |
| クロック周期 | 10ns (100MHz基準) |

### 計測方法

テストベンチ `tb_jpeg_perf.sv` でグラデーション画像をオンザフライ生成し、以下の4区間のクロック数を計測:

- **入力フェーズ**: 最初のピクセル入力ハンドシェイク → 最後のピクセル入力ハンドシェイク
- **パイプラインレイテンシ**: 最初のピクセル入力 → 最初のJPEGバイト出力
- **出力フェーズ**: 最初のJPEGバイト出力 → 最後のJPEGバイト出力 (EOI)
- **合計エンコード**: 最初のピクセル入力 → 最後のJPEGバイト出力 (EOI)

## 4. スループット分析

### 4.1 ピクセルあたりクロック数

#### 4:4:4 モード

全サイズで **約54-58 clocks/pixel** (0.017-0.019 pixels/clock) と安定。

```
clocks/pixel = 合計エンコード clocks / 総ピクセル数

  8x8:       58.4 clocks/pixel
 16x16:      55.7 clocks/pixel
 32x32:      54.7 clocks/pixel
 64x64:      54.4 clocks/pixel
128x128:     54.2 clocks/pixel
256x256:     54.2 clocks/pixel
640x480:     55.2 clocks/pixel
1280x720:    55.7 clocks/pixel
1920x1080:   55.8 clocks/pixel
```

#### 4:2:0 モード

全サイズで **約28-30 clocks/pixel** (0.034-0.036 pixels/clock) と安定。

```
 16x16:      29.4 clocks/pixel
 32x32:      28.1 clocks/pixel
 64x64:      28.3 clocks/pixel
128x128:     28.6 clocks/pixel
256x256:     28.8 clocks/pixel
640x480:     29.0 clocks/pixel
1280x720:    29.0 clocks/pixel
```

4:2:0は4:4:4比で**約1.9倍高速**。16x16 MCUあたり6ブロック (Y×4 + Cb + Cr) を処理するが、4:4:4が同じ16x16領域で12ブロック (Y×4 + Cb×4 + Cr×4) を必要とするため、処理ブロック数が半分になる。

### 4.2 ボトルネック分析

#### 4:4:4 モード (1ピクセルあたり約55クロック)

| 処理 | 推定clocks/pixel | 説明 |
|------|------------------|------|
| rgb2ycbcr | ~1 | 3段パイプライン、1 pixel/clock |
| block_splitter | ~1 | ラインバッファ書込/読出 |
| Component Mux | ~3 | 3成分順次出力 (1ブロック64pix × 3成分 / 64pix) |
| dct_2d | ~16 | 64入力 → 行DCT → 転置 → 列DCT → 64出力 × 3成分 |
| quantizer | ~3 | 除算パイプライン × 3成分 |
| zigzag_scan | ~6 | バッファ書込/読出 × 3成分 |
| rle_encoder | ~3 | 可変長出力 × 3成分 |
| huffman_encoder | ~3 | 可変長出力 × 3成分 |
| bitstream_assembler | ~3 | ビットパッキング |
| バックプレッシャ | ~16 | パイプラインストール |
| **合計** | **~55** | |

#### 4:2:0 モード (1ピクセルあたり約29クロック)

| 処理 | 推定clocks/pixel | 説明 |
|------|------------------|------|
| rgb2ycbcr | ~1 | 3段パイプライン、1 pixel/clock |
| block_splitter_420 | ~2 | 16行ラインバッファ + ダウンサンプリング |
| Component Mux | ~1.5 | 6ブロック/MCU (Y4+Cb+Cr) vs 16x16=256pix |
| dct_2d | ~8 | 6ブロック/MCU (4:4:4の12ブロックの半分) |
| quantizer | ~1.5 | 6ブロック × 除算パイプライン |
| zigzag_scan | ~3 | 6ブロック × バッファ書込/読出 |
| rle_encoder | ~1.5 | 6ブロック × 可変長出力 |
| huffman_encoder | ~1.5 | 6ブロック × 可変長出力 |
| bitstream_assembler | ~1.5 | ビットパッキング |
| バックプレッシャ | ~7.5 | パイプラインストール (4:4:4の約半分) |
| **合計** | **~29** | |

主なボトルネック:
- **DCT演算**: 行・列それぞれ8x8行列乗算が必要で、最も処理時間がかかる
- **3成分順次処理**: 共有パイプラインにY→Cb→Crを順次投入するため、実効スループットが制限される
- **バッファリング**: block_splitter、zigzag_scan、dct_2dがそれぞれブロック単位のバッファリングを要求
- **4:2:0の効果**: クロマダウンサンプリングによりDCTブロック数が半減し、スループットが約1.9倍に向上

## 6. 高速化の方向性

現行設計は**共有パイプライン方式**（Y/Cb/Crを1本のパイプラインで順次処理）のため、3成分モードではスループットが理論値の1/3に制限される。以下の最適化により大幅な高速化が可能:

### 6.1 パイプライン並列化 (3倍高速化)
- Y/Cb/Cr各成分に独立したDCT→量子化→ジグザグ→RLE→Huffmanパイプラインを配置
- 3成分を完全並列処理し、スループットを3倍に
- 推定: ~18 clocks/pixel → 1920x1080 @200MHz で約5.4 fps

### 6.2 DCT高速化 (2-4倍高速化)
- 現行: 直接行列乗算（8x8 = 64回の乗算累積）
- 改善案1: バタフライ演算による高速DCT (Loeffler算法等)
- 改善案2: 完全パイプライン化DCT（1 clock/coefficient入力）
- 推定: DCT部分の処理時間を1/2～1/4に短縮

### 6.3 入力並列化 (2倍高速化)
- 現行: 1 pixel/clock入力
- 改善: 2 pixel/clock または 4 pixel/clock 入力対応
- AXI4-Stream データ幅を64-bit/128-bitに拡張

### 6.4 4:2:0クロマサブサンプリング (1.9倍高速化) ✅ 実装済み
- 4:4:4: 各ピクセルのY/Cb/Crを全てフル解像度で処理 → 3ブロック/8x8MCU
- 4:2:0: Cb/Crを2x2ダウンサンプリング → 6ブロック/16x16MCU (同面積で半分)
- 実測: 55 clk/pix (4:4:4) → 29 clk/pix (4:2:0) = **1.9倍高速**
- 圧縮率も向上: 1280x720で 99.3:1 → 161.9:1 (JPEG出力が約39%小型化)

### 6.5 複合最適化の効果推定

| 最適化 | 推定 clocks/pixel | 1920x1080 @200MHz |
|--------|-------------------|-------------------|
| 現行設計 (4:4:4) | ~55 | 1.73 fps |
| **4:2:0モード** | **~29** | **~3.3 fps** ✅ |
| パイプライン並列化 (4:4:4) | ~18 | 5.4 fps |
| パイプライン並列化 (4:2:0) | ~10 | ~9.6 fps |
| + DCT高速化 | ~5 | ~19.3 fps |
| + 入力並列化 | ~2.5 | ~38.6 fps |
| 全最適化 (4:2:0) | ~2.5 | @250MHz で ~48 fps |
